<p>Direktori <code>lib</code> adalah jantung dari aplikasi Flutter Anda, berisi semua kode Dart yang menjalankan aplikasi. Proyek ini tampaknya mengikuti struktur yang rapi (mirip dengan <i>Clean Architecture</i>) yang memisahkan antara logika, data, dan tampilan.</p>

<hr>

<h3><code>lib/</code> (Folder Root)</h3> <ul> <li><strong><code>main.dart</code></strong>: Ini adalah titik masuk (<i>entry point</i>) utama aplikasi Anda. File ini bertanggung jawab untuk menginisialisasi hal-hal penting seperti Firebase dan menjalankan aplikasi.</li> <li><strong><code>firebase_options.dart</code></strong>: File ini dibuat otomatis oleh Firebase. Isinya adalah semua kunci konfigurasi (API keys, project ID) yang diperlukan aplikasi Anda untuk terhubung ke layanan Firebase (seperti Firestore) di berbagai platform (Android, iOS, Web).</li> </ul>

<hr>

<h3><code>lib/application/</code></h3> <p>Folder ini berisi logika bisnis inti aplikasi Anda. Ini adalah "otak" yang mengelola status (state) aplikasi.</p> <ul> <li><strong><code>notifiers/</code></strong>: Folder ini berisi kelas-kelas <i>StateNotifier</i> dari Riverpod. <ul> <li><strong><code>home_notifier.dart</code></strong>: Mengelola status untuk halaman utama, termasuk daftar resep dari Firestore, filter pencarian, dan fungsi untuk menambah, mengedit, atau menghapus resep di Firestore.</li> <li><strong><code>saved_recipes_notifier.dart</code></strong>: Mengelola status untuk resep favorit, seperti memuat, menyimpan, atau menghapus resep dari penyimpanan lokal.</li> <li><strong><code>scan_notifier.dart</code></strong>: Mengelola status untuk proses pemindaian. Ini mencakup logika untuk mengambil gambar, mengirimkannya ke API eksternal di Hugging Face, dan menyimpan hasil deteksi bahan makanan.</li> </ul> </li> <li><strong><code>providers/</code></strong>: Folder ini mendefinisikan <i>Provider</i> Riverpod, yang "menyediakan" Notifier ke UI (tampilan). <ul> <li><strong><code>home_provider.dart</code></strong>: Menyediakan <code>HomeNotifier</code> agar bisa diakses oleh halaman utama.</li> <li><strong><code>saved_recipes_provider.dart</code></strong>: Menyediakan <code>SavedRecipesNotifier</code> dan <code>RecipeLocalRepository</code>.</li> <li><strong><code>scan_provider.dart</code></strong>: Menyediakan <code>ScanNotifier</code> agar bisa diakses oleh halaman pindai.</li> </ul> </li> </ul>

<hr>

<h3><code>lib/core/</code></h3> <p>Folder ini berisi kode utilitas umum yang dapat digunakan di seluruh bagian aplikasi.</p> <ul> <li><strong><code>constants/app_colors.dart</code></strong>: Menyimpan palet warna utama aplikasi (seperti warna primer, sekunder, error) sebagai konstanta agar mudah digunakan kembali.</li> <li><strong><code>utils/responsive.dart</code></strong>: File utilitas (pembantu) untuk menangani desain responsif, misalnya dengan mengecek apakah perangkat yang digunakan adalah mobile, tablet, atau desktop.</li> </ul>

<hr>

<h3><code>lib/domain/</code></h3> <p>Folder ini mendefinisikan model data inti dari aplikasi Anda.</p> <ul> <li><strong><code>models/recipe.dart</code></strong>: Mendefinisikan struktur data (model) untuk sebuah <code>Recipe</code> (Resep). Ini menentukan properti apa saja yang dimiliki resep (seperti id, judul, durasi, bahan, langkah) dan menyertakan fungsi untuk mengubah data dari/ke format JSON.</li> </ul>

<hr>

<h3><code>lib/infrastructure/</code></h3> <p>Folder ini menangani sumber data (dari mana data berasal), seperti database lokal atau API eksternal.</p> <ul> <li><strong><code>local/recipe_local_repository.dart</code></strong>: Ini adalah "Repositori" yang mengurus logika penyimpanan data resep favorit secara lokal di perangkat pengguna menggunakan <code>SharedPreferences</code>.</li> </ul>

<hr>

<h3><code>lib/presentation/</code></h3> <p>Folder ini berisi semua kode yang terkait dengan Tampilan (UI) atau apa yang dilihat oleh pengguna.</p> <ul> <li><strong><code>routes/app_router.dart</code></strong>: Mengkonfigurasi semua rute navigasi (perpindahan antar halaman) di aplikasi Anda menggunakan package <code>GoRouter</code>.</li> <li><strong><code>screens/</code></strong>: Berisi file-file untuk setiap halaman/layar penuh pada aplikasi. <ul> <li><strong><code>add_edit_recipe_screen.dart</code></strong>: Halaman formulir yang digunakan pengguna untuk menambah resep baru atau mengedit resep yang sudah ada.</li> <li><strong><code>home_screen.dart</code></strong>: Halaman utama (beranda) aplikasi. Ini menampilkan daftar resep, tombol filter, dan bilah navigasi bawah (Bottom Navigation Bar).</li> <li><strong><code>recipe_detail_screen.dart</code></strong>: Halaman yang menampilkan informasi rinci dari satu resep, termasuk gambar, daftar bahan, dan langkah-langkah pembuatannya.</li> <li><strong><code>saved_recipes_screen.dart</code></strong>: Halaman yang menampilkan daftar resep yang telah disimpan (difavoritkan) oleh pengguna.</li> <li><strong><code>scan_screen.dart</code></strong>: Halaman untuk fitur pemindaian bahan. Pengguna dapat memilih gambar dari galeri atau kamera, dan melihat hasil deteksi bahan.</li> </ul> </li> <li><strong><code>widgets/</code></strong>: Berisi komponen-komponen UI kecil yang dapat digunakan kembali di berbagai halaman. <ul> <li><strong><code>empty_state.dart</code></strong>: Widget yang ditampilkan di tengah layar ketika tidak ada data untuk ditampilkan (misalnya, saat daftar favorit kosong).</li> <li><strong><code>filter_bottom_sheet.dart</code></strong>: Widget yang muncul dari bawah layar (Bottom Sheet) dan berisi semua opsi filter (durasi, kesulitan, kategori, dll.) untuk halaman utama.</li> <li><strong><code>recipe_card.dart</code></strong>: Widget kustom berbentuk kartu yang digunakan untuk menampilkan ringkasan resep (gambar, judul, durasi) di halaman utama dan halaman favorit.</li> </ul> </li> </ul>
